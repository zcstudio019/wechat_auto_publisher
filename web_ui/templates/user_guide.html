操作流程
共 6 步
1
登录系统

打开浏览器访问系统地址，使用分配的账号密码登录。

地址：http://你的IP:5000
用户名：editor   密码：editor123
2
创建新文章

进入「文章管理」→ 点击右上角「+ 新建文章」按钮。

点击「+ 新建文章」
填写标题和正文
选择内容分类标签
3
使用写作中心（推荐）

进入「配置中心」→「写作中心」→ 选择对应分类模板 → 点击「一键写作」。

提示：系统提供 6 大类写作中心——自动获客、品牌宣传、知识科普、方案匹配、融资规划、企业经营分析。使用模板可自动生成结构化高质量内容。
4
AI 格式化处理

在文章详情页 → 点击「格式化」按钮 → 系统自动完成：

  • Markdown 转 HTML 排版
  • 关键词高亮（利率、额度等数字）
  • 敏感词检测与替换
  • 自动生成配图卡片（场景图/金句卡/数据卡/贴士卡）
  • 生成「简单说」摘要块
5
提交审核

确认内容无误后，在文章操作栏中点击「提交审核」按钮。

注意：提交后文章将锁定，无法自行修改。如需修改需联系运营人员退回。
6
查看数据效果

进入「数据分析报表」页面，查看：

内容效果
阅读量、转发数、获客数、转化率
转化漏斗
阅读→表单→留资→线索→成交 全链路

支持按时间范围（7天/30天/90天）和内容类型筛选。

常见问题
进入左侧菜单「发布管理」→「文章列表」，所有文章按状态分类显示（草稿/待审/已发布）。也可以在顶部搜索框输入关键词快速查找。
根据文章主题选择：

A. 文章审核
每日必做
  1. 进入待审队列
    「文章管理」→ 筛选状态=「待审核」
  2. 阅读检查
    点击文章查看详情，检查：标题是否吸引、内容是否准确、是否有错别字、配图是否合适
  3. 做出决定
    ✅ 「通过」— 自动推送到微信草稿箱
    ❌ 「拒绝」— 填写原因后退回给编辑修改
  4. 关注推送结果
    通过后会显示推送状态。若微信API异常需手动重试「推送到微信」
审核要点：
  1. 确保无政治敏感词
  2. 金融数据必须标注来源
  3. 不得承诺具体利率或放款时效
B. 线索管理
实时跟进
  1. 查看新线索
    「业务模块」→「线索列表」→ 筛选状态=「新建」
  2. 确认分配情况
    系统已按客户地区自动分配顾问。可在「分配」列查看或手动重新分配
  3. 更新线索状态
    新建 → 已分配 → 已联系 → 已转化 / 已流失
  4. 查看关联信息
    点击线索可看到来源文章、提交的详细表单数据
数据联动：线索来自文章页面的留资表单，每条线索都记录了来源文章ID，可追踪哪篇文章获客效果最好。
C. 工单处理
按时响应
  1. 查看待处理工单
    「业务模块」→「工单列表」→ 筛选状态=「待处理」
  2. 工单类型识别
    🔵 贷款方案匹配
    🟢 融资规划
    🟠 企业经营分析
  3. 分配/接单
    系统已根据类型匹配专长顾问，也可手动调整
  4. 更新进度
    待处理 → 处理中 → 已完成
  5. 添加交付物
    上传方案报告/文档作为交付记录
D. 数据报表
周报用
  1. 实时数据看板
    「数据分析报表」顶部状态栏实时显示：待审稿件数、今日新增线索、待处理工单数
  2. 内容效果分析
    各类型文章的阅读量、转发数、获客转化率
  3. 服务交付统计
    各类工单数量、完成率、平均评分
  4. 导出报告
    支持 Excel 和 PDF 两种格式导出
  5. 预警监控
    「预警配置」页面设置线索下降/工单超时告警规则

C. 安全与运维

密钥管理

  • 微信公众号 AppID/AppSecret 在 config.py 中配置
  • 微信 IP 白名单需包含服务器公网 IP
  • 定期更换密钥（建议90天）

数据库维护

  • 数据库路径：data/articles.db（SQLite）
  • 建议定期备份：复制整个 data/ 目录
  • 不要直接编辑 DB 文件，通过系统界面操作

服务运行

  • 启动命令：python main.py（从项目根目录）
  • 默认端口：5000
  • 日志输出在控制台
D. 常见问题排查
  1. 检查 IP 白名单：微信公众平台 → 开发 → 基本配置 → IP白名单
  2. 检查 AppID/AppSecret 是否正确
  3. 检查服务器网络能否访问 api.weixin.qq.com
  1. 确认从项目根目录启动（不是子目录）
  2. 检查 Python 版本 >= 3.8
  3. 执行 python -c "from web_ui.app import app; print('OK')" 测试导入
  4. 查看控制台完整 Traceback 错误信息
